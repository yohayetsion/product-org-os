# AXIA V3 Presentation Template Reference

**Single source of truth for the HTML presentation template system.**

---

## 1. Overview

- All HTML presentations MUST use the AXIA V3 slide-based template (NOT reveal.js)
- Generated by `agent-output-handler.py` via the `MOBILE_HTML_TEMPLATE` string constant
- Self-contained: no external dependencies (no CDN, no reveal.js)
- Every content block supports inline commenting out of the box
- Output: single `.html` file in `C:\dev\presentations\`

---

## 2. Architecture

### Template: AXIA V3 Slide-Based Format

**Theme:**
- Background: `#0D1117` (dark)
- Card surfaces: `#161B22`
- Elevated surfaces: `#1C2333`
- Border: `#30363D`
- Text primary: `#E6EDF3`
- Accent cyan: `#22D3EE`
- Brand gradient: `linear-gradient(135deg, {primary_color}, #22D3EE)`

**Typography:**
- Headings: `Manrope` (Google Fonts, weight 400-800)
- Code: `JetBrains Mono` (Google Fonts)
- Base size: `15px`, line-height `1.7`

**Slide Structure:**
- Markdown split by `## ` (h2) headers; each becomes one slide
- Slides use `display: none` / `.active { display: flex }` toggling
- `.slide { min-height: 100vh; overflow-y: auto; }` -- content MUST scroll, NEVER clip
- Each slide contains a brand bar (logo mark + logo text + section label) at the top
- Content blocks wrapped in `.commentable` divs with `data-section` and `data-label` attributes
- Grid background via `::before` pseudo-element on `.slide`

**Critical CSS rule:**
```css
/* CORRECT - scrollable */
.slide { min-height: 100vh; overflow-y: auto; }

/* WRONG - clips content */
.slide { height: 100vh; overflow: hidden; }
```

### Navigation

**CSS Grid nav bar** (fixed bottom, `z-index: 100`):
```
[section-dots]  |  [prev] [counter] [next]  |  [slide-dots]
```
- Three-column grid: `grid-template-columns: 1fr auto 1fr`
- Background: `rgba(13,17,23,0.95)` with `backdrop-filter: blur(12px)`
- Section dots (left): colored dots per h2 group, click to jump to group start
- Slide dots (right): one dot per slide, active dot highlighted, dots with comments marked
- Counter: displays `{current} / {total}`, clickable to open jump overlay

**Keyboard:**
| Key | Action |
|-----|--------|
| Left arrow | Previous slide |
| Right arrow | Next slide |
| Home | First slide |
| End | Last slide |
| Space | Next slide |
| Escape | Close jump overlay |

**Touch:** Swipe left/right via `touchstart`/`touchend` handlers (threshold: 50px).

**Slide jump overlay:** Click the counter (e.g., "5 / 37") to open a modal with a number input. Enter to jump, Escape to close. Styled to match dark theme.

**Hidden scrollbar:**
```css
html { scrollbar-width: none; }
html::-webkit-scrollbar { display: none; }
```

### Brand Detection

Auto-detected from the presentation title via `_detect_brand(title)`. Sets CSS `--primary`, favicon, logo letter, and logo text.

| Brand | Match | Color | Favicon | Logo |
|-------|-------|-------|---------|------|
| AXIA | `axia` in title | `#0D9488` (teal) | "A" on teal | `A` / `AXIA` |
| Legionis | `legionis` in title | `#6366F1` (indigo) | "L" on indigo | `L` / `Legionis` |
| SKYMOD | `skymod` in title | `#C4A962` (gold) | "S" on gold | `S` / `SKYMOD` |
| Maad House | `maad` in title | `#E95420` (orange) | "M" on orange | `M` / `Maad House` |

Favicons are inline SVG data URIs (no external files).

### Inline Commenting (Always-On)

Commenting is built into every presentation. Not optional, not a separate layer.

**Structure:**
- Every content block (headings, paragraphs, tables, lists, blockquotes) wrapped in `.commentable` divs
- Each div has `data-section="s{slide}-{block}"` (unique ID) and `data-label="..."` (display text)
- `+` button appears via CSS `::after` pseudo-element on hover (desktop) or always faintly visible (touch devices via `@media (hover: none)`)

**Comment flow:**
1. Click `+` on any content block
2. Fixed-position popover opens with textarea
3. Save persists to `localStorage` (key: `feedback_{sanitized_title}`)
4. Block gets `.has-comment` class (left border highlight + dot indicator)
5. Corresponding slide dot in nav bar gets `.has-comment` class

**Review slide (always last):**
- Auto-appended as the final slide
- Lists all comments by section with Edit/Remove buttons
- General feedback textarea
- Export options: Markdown or JSON, copy to clipboard or download as file

**Mobile handling:**
- Popover anchors to bottom of screen (`bottom: 60px`) on touch devices
- `+` buttons always faintly visible (not hover-dependent)

### IIFE Scope

Two script blocks in the template:
1. **Navigation IIFE** -- slide show/hide, keyboard/touch handlers, section groups
2. **Commenting IIFE** -- comment storage, popover, review slide rendering

Between them: the comment popover HTML (injected as a DOM string).

All functions called from inline `onclick` handlers MUST be exposed via `window.`:
```js
window.showSlide = showSlide;
window.promptSlide = function() {...};
window.closeJump = function() {...};
window.showComment = showComment;
window.saveComment = saveComment;
window.closePopover = closePopover;
window.editComment = editComment;
window.removeComment = removeComment;
window.exportFeedback = exportFeedback;
window.downloadFeedback = downloadFeedback;
```

## 3. How Presentations Are Generated

### Command
```bash
python "G:\My Drive\Claude\agent-output-handler.py" "<path-to-md-file>" --no-telegram
```

### Process
1. Reads the markdown file
2. Splits content by `## ` headers into sections (each = one slide)
3. Calls `_detect_brand(title)` to determine colors, favicon, logo
4. Converts each section's markdown to HTML via `convert_md_to_html()`
5. Splits HTML into content blocks via `_split_html_into_blocks()`
6. Wraps each block in `.commentable` div with `data-section` and `data-label`
7. Builds slide HTML with brand bar per slide
8. Tracks section groups (h2 headings) for section dot navigation
9. Formats `MOBILE_HTML_TEMPLATE` with all generated content
10. Saves to `C:\dev\presentations\{slugified-title}.html`
11. Opens in default browser

### Flags
| Flag | Effect |
|------|--------|
| `--no-telegram` | Skip Telegram notification (ALWAYS use this) |
| `--no-browser` | Don't auto-open in browser |
| `--json` | Output result as JSON |
| `--commentable` | Legacy flag (commenting is now always-on) |

## 4. Key Implementation Rules

### 4.1 Scrollable Slides
```css
.slide { min-height: 100vh; overflow-y: auto; }
```
NEVER use `height: 100vh; overflow: hidden`. Dense content (tables, action lists) must scroll.

### 4.2 Python Template Escaping
The `MOBILE_HTML_TEMPLATE` is a Python f-string-compatible template using `.format()`:
- `{{` and `}}` for literal CSS/JS braces (Python escaping)
- `\\n` for JS newline characters in string literals
- `\\\'` for JS single quotes inside innerHTML strings
- Template variables: `{title}`, `{favicon}`, `{primary_color}`, `{logo_letter}`, `{logo_text}`, `{slides_html}`, `{sections_json}`, `{num_sections}`, `{num_slides_display}`

### 4.3 No External Dependencies
Everything is inline:
- CSS: embedded in `<style>` block
- JS: embedded in `<script>` blocks
- Fonts: Google Fonts `<link>` (Manrope + JetBrains Mono) -- the only external request
- Favicons: inline SVG data URIs

### 4.4 Content Z-Index
- Slide `::before` grid background: default stacking
- Content elements inside slides: `position: relative; z-index: 1`
- Comment `::after` buttons: `z-index: 5`
- Nav bar: `z-index: 100`
- Jump overlay: `z-index: 150`
- Comment popover: `z-index: 150`
- Toast notifications: `z-index: 200`

### 4.5 Two Script Blocks
Order in the HTML:
1. Navigation IIFE (`<script>`)
2. Comment popover HTML (injected via `document.body.insertAdjacentHTML`)
3. Commenting IIFE (`<script>`)

Both IIFEs are self-contained. Navigation fires first, then commenting initializes and reads localStorage.
