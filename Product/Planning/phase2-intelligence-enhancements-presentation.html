Feature 1: ROI Tracker

Per-Interaction Display (Claude Code Style)

  • Time saved after EVERY skill/agent
  • Session running total visible
  • Interaction-level storage
  • Dashboard with 30/90 day views
  • Export PDF/CSV for stakeholders
  • Quiet mode for power users

Example output:

[PRD created successfully]
---

Feature 2: Intelligent Routing

Orchestrator Pattern

  • Analyze request before any spawn
  • Determine MINIMUM agents needed
  • Generate rich guidance per agent
  • Agents can spawn sub-agents
  • Sub-agents inherit context
  • NO questions (unless truly ambiguous)

Philosophy shift:

OLD: "This could be X or Y. Which do you want?"

NEW: System picks the best option, executes, user can override if wrong.

Orchestrator Flow

User Request
v
Orchestrator
Parse intent | Select minimum agents | Generate guidance
v
Primary Agent
(with guidance)
Sub-Agent
(if needed)
v
Result + ROI Display
