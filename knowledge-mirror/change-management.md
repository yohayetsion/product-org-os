## Change Management Frameworks

### ADKAR Model (Prosci)

| Phase | Focus | Key Question | Tactics |
|-------|-------|-------------|---------|
| **A**wareness | Why the change is needed | "Do people understand the reason?" | Business case communication, executive sponsorship, data sharing |
| **D**esire | Personal motivation to support | "Do people want to participate?" | WIIFM messaging, resistance management, incentive alignment |
| **K**nowledge | How to change | "Do people know what to do differently?" | Training programs, job aids, coaching |
| **A**bility | Demonstrated capability | "Can people perform the new way?" | Practice, hands-on support, time to build proficiency |
| **R**einforcement | Sustained adoption | "Will people stick with it?" | Recognition, accountability, feedback loops, course correction |

**Usage**: Diagnose where individuals are stuck. If adoption stalls, identify which ADKAR element is the barrier -- awareness gaps need communication, desire gaps need motivation, etc.

### Kotter's 8-Step Model

```
CREATING CLIMATE FOR CHANGE
  1. Create Urgency        — Build a compelling case for why change must happen now
  2. Form a Guiding Coalition — Assemble a group with enough power and credibility to lead
  3. Create a Vision        — Develop a clear, compelling picture of the future state

ENGAGING & ENABLING THE ORGANIZATION
  4. Communicate the Vision — Use every channel, repeatedly, to build understanding
  5. Empower Action         — Remove barriers, change systems that undermine the vision
  6. Generate Short-Term Wins — Plan and create visible improvements early

IMPLEMENTING & SUSTAINING
  7. Consolidate Gains      — Use credibility from wins to drive more change
  8. Anchor in Culture      — Connect new behaviors to organizational success
```

**Common failure**: Skipping Step 1 (urgency). Without urgency, Step 2-8 face passive resistance. Another frequent failure: declaring victory too early (after Step 6) before changes are anchored in culture.

### Lewin's Change Model

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   UNFREEZE   │ ──> │    CHANGE    │ ──> │   REFREEZE   │
│              │     │  (Transition)│     │              │
│ - Challenge  │     │ - Implement  │     │ - Stabilize  │
│   status quo │     │   new ways   │     │   new state  │
│ - Create     │     │ - Support    │     │ - Reinforce  │
│   motivation │     │   through    │     │   new norms  │
│ - Reduce     │     │   uncertainty│     │ - Update     │
│   resistance │     │ - Learn new  │     │   systems &  │
│              │     │   behaviors  │     │   processes  │
└──────────────┘     └──────────────┘     └──────────────┘
```

**Key insight**: Most change efforts focus on the "Change" phase and neglect Unfreeze (people aren't ready) or Refreeze (changes don't stick). Unfreeze typically requires 2-3x more effort than organizations expect.

### McKinsey 7-S Framework

```
                    ┌──────────────────┐
                    │  Shared Values   │
                    │   (center of     │
                    │    alignment)    │
                    └────────┬─────────┘
           ┌────────────┬────┴───┬────────────┐
     ┌─────┴─────┐ ┌───┴────┐ ┌┴─────┐ ┌────┴─────┐
     │ Structure │ │Strategy│ │Systems│ │  Style   │
     │ (hard)    │ │ (hard) │ │(hard) │ │ (soft)   │
     └───────────┘ └────────┘ └──────┘ └──────────┘
           ┌────────────┬────────────┐
     ┌─────┴─────┐ ┌───┴────┐ ┌────┴─────┐
     │  Skills   │ │ Staff  │ │          │
     │ (soft)    │ │ (soft) │ │          │
     └───────────┘ └────────┘ └──────────┘
```

| Element | Description | Change Implication |
|---------|-------------|-------------------|
| **Strategy** | Plan for competitive advantage | Does the change align with strategic direction? |
| **Structure** | Org chart, reporting lines | Does the structure need to change to support new ways? |
| **Systems** | Processes, IT, workflows | Which systems must be updated? |
| **Shared Values** | Core beliefs, culture | Does the change conflict with cultural values? |
| **Style** | Leadership and management style | Do leaders model the desired behaviors? |
| **Staff** | People, talent, capabilities | Do we have the right people and skills? |
| **Skills** | Organizational competencies | What new skills must be developed? |

**Usage**: When planning change, assess all 7 elements. Misalignment in any one can derail the initiative. Hard elements (Strategy, Structure, Systems) are easier to change; soft elements (Shared Values, Style, Staff, Skills) take longer but are more critical for sustained adoption.

### Bridges' Transition Model

```
EVENT (the change)
  │
  ▼
┌─────────────────────────────────────────────────────────┐
│                    TRANSITION                           │
│                                                         │
│  Phase 1: ENDING         Phase 2: NEUTRAL    Phase 3:   │
│  (Letting Go)            ZONE               NEW         │
│                                             BEGINNING   │
│  - Acknowledge loss      - Confusion        - Energy    │
│  - Grieve what's gone    - Creativity       - Clarity   │
│  - Mark the ending       - Innovation       - Commitment│
│  - Respect the past      - Anxiety          - New       │
│                          - Experimentation    identity  │
└─────────────────────────────────────────────────────────┘
```

**Key distinction**: Change is external (new system, reorg, merger). Transition is internal (psychological reorientation). Change can happen quickly; transition takes time. Ignoring the emotional transition is the #1 reason technically sound changes fail.

**The Neutral Zone** is the most dangerous phase -- old ways are gone but new ways aren't yet comfortable. Productivity dips. Anxiety rises. Leaders must provide extra support, temporary structures, and quick wins during this period.

## Communication Planning

### Change Communication Plan Template

| Phase | Timing | Audience | Key Message | Channel | Sender | Feedback Mechanism |
|-------|--------|----------|-------------|---------|--------|-------------------|
| Awareness | Week 1-2 | All employees | Why we are changing, what it means | Town hall + email | CEO/Sponsor | Q&A session |
| Awareness | Week 1-2 | Managers | Their role in the change, talking points | Manager briefing | Change lead | Manager feedback form |
| Understanding | Week 3-4 | Affected teams | What changes specifically, timeline | Team meetings | Direct managers | Team discussions |
| Engagement | Week 5-8 | Change champions | Deep dive, equip to support peers | Champion workshop | Change team | Champion network |
| Enablement | Week 6-10 | End users | How to do things the new way | Training sessions | Trainers | Skill assessments |
| Reinforcement | Ongoing | All affected | Progress, wins, adjustments | Newsletter + dashboards | Change team | Pulse surveys |

### Message Framing by Audience

| Audience | Care About | Frame As | Avoid |
|----------|-----------|----------|-------|
| **Executives** | Strategic alignment, ROI, risk, competitive position | Business case, strategic enabler, market driver | Operational details, training logistics |
| **Senior Managers** | Resource needs, timeline, team impact, their role | "You are critical to success," empowerment, decision rights | Implying they are obstacles, bypassing them |
| **Middle Managers** | Day-to-day disruption, team morale, workload | Practical impact, support available, phased approach | Minimizing difficulty, adding accountability without authority |
| **Front-line Staff** | "What changes for me?", job security, daily work | Personal benefit (WIIFM), training, support, timeline | Corporate jargon, vague reassurance, "exciting opportunity" |
| **External (customers/partners)** | Service continuity, improvement, timeline | Better experience, enhanced capability, planned transition | Internal politics, organizational reasons |

### Communication Channel Selection Matrix

| Channel | Best For | Reach | Richness | Frequency |
|---------|----------|-------|----------|-----------|
| Town hall / All-hands | Major announcements, vision, Q&A | High | High | Quarterly or at milestones |
| Email / memo | Formal updates, documentation, reference | High | Low | As needed (don't overuse) |
| Manager cascade | Team-specific impact, personal discussion | Medium | High | Weekly during transition |
| Intranet / wiki | Self-service info, FAQ, resources | High | Medium | Continuously updated |
| Slack / Teams channels | Quick updates, questions, community | Medium | Medium | Daily/as needed |
| 1-on-1 meetings | High-impact individuals, resistance | Low | Very High | As needed |
| Video / recorded message | Consistent message, emotional tone | High | Medium | Key milestones |
| Dashboards | Progress tracking, metrics | Medium | Low | Real-time or weekly |

### FAQ Development Framework

```
STRUCTURE FOR CHANGE FAQ

1. THE BASICS
   - What is changing?
   - Why is this change happening?
   - Why now?
   - Who decided this?

2. PERSONAL IMPACT
   - How does this affect my day-to-day work?
   - Will my role change?
   - Will my team change?
   - Will my compensation/benefits be affected?

3. TIMELINE & PROCESS
   - When does this happen?
   - What are the key milestones?
   - Will there be a pilot/phased rollout?
   - Can I continue the old way during transition?

4. SUPPORT & RESOURCES
   - What training will be provided?
   - Who do I contact with questions?
   - Where can I find more information?
   - What support is available during transition?

5. INPUT & FEEDBACK
   - How can I provide input?
   - Will my feedback be considered?
   - How will decisions be communicated?
   - What if I disagree with the approach?

GUIDELINES:
- Write in plain language (no corporate jargon)
- Be honest about what is unknown (say "we don't know yet" rather than dodging)
- Update regularly as new questions emerge
- Include the date of last update
```

| Keep Satisfied | Address concerns proactively, demonstrate respect | Regular updates, responsive to requests | Informed and consulted |

### Resistance Identification and Mitigation

| Resistance Type | Signals | Root Cause | Mitigation Strategy |
|-----------------|---------|-----------|---------------------|
| **Rational** | Questions about evidence, asks for data, challenges logic | Genuine concerns about feasibility or impact | Provide data, pilot results, address specific concerns with facts |
| **Emotional** | Anxiety, nostalgia for "old ways", frustration | Fear of loss (status, comfort, competence, relationships) | Acknowledge loss, provide psychological safety, demonstrate support |
| **Political** | Passive-aggressive compliance, back-channel criticism | Threat to power base, territory, or influence | Involve early, give meaningful role, address power concerns directly |
| **Cultural** | "That's not how we do things here", value conflicts | Clash with organizational identity or norms | Connect to existing values, reframe change as evolution not revolution |
| **Capacity** | "We're too busy", missed deadlines, poor quality | Genuine overload or lack of skills | Reduce workload, extend timeline, provide resources and training |

**Key principle**: Resistance is information, not obstruction. Every form of resistance reveals something important about the change or the organization. Listen before solving.

### Champion / Change Agent Network Design

```
NETWORK STRUCTURE

Executive Sponsor(s)
  └── Change Leadership Team (3-5 senior leaders)
        └── Change Champions (1 per 15-25 affected employees)
              └── Peer Supporters (informal, organic)

CHAMPION SELECTION CRITERIA:
- Respected by peers (credibility > seniority)
- Positive but realistic (not blind optimists)
- Connected across teams (informal influencers)
- Willing to give honest feedback upward
- Available to dedicate [TBD]% of time

CHAMPION RESPONSIBILITIES:
1. Model new behaviors visibly
2. Listen to concerns and relay themes upward
3. Translate corporate messaging into team context
4. Provide peer coaching and support
5. Celebrate early wins and progress
6. Flag issues early before they become crises

SUPPORT FOR CHAMPIONS:
- Monthly champion meetings (share experiences, problem-solve)
- Early access to information (always know before their teams)
- Direct line to change leadership team
- Recognition for contribution
- Training on coaching and facilitation skills
```

### Sponsor Roadmap

| Phase | Sponsor Actions | Time Investment | Support Needed |
|-------|----------------|-----------------|----------------|
| **Build Urgency** | Communicate business case, share data, tell the story | High (visible, frequent) | Data package, talking points, executive coaching |
| **Form Coalition** | Recruit peers, align leadership team, resolve disagreements | High (1-on-1 meetings) | Stakeholder analysis, meeting facilitation |
| **Launch** | Kick off publicly, demonstrate personal commitment | High (visible presence) | Communications, event planning |
| **Sustain** | Remove barriers, recognize progress, address resistance | Medium (regular check-ins) | Progress reports, escalation protocols |
| **Reinforce** | Celebrate wins, connect to business results, embed in systems | Medium (strategic moments) | Metrics, success stories |

**Sponsor anti-pattern**: The "absent sponsor" who launches the change and disappears. Sponsorship requires sustained, visible engagement throughout the entire initiative.

## Training and Enablement

### Training Needs Assessment

```
STEP 1: DEFINE FUTURE STATE REQUIREMENTS
  For each role affected by the change:
  - What tasks will they perform differently?
  - What new tools/systems will they use?
  - What new decisions will they make?
  - What old tasks/tools are eliminated?

STEP 2: ASSESS CURRENT STATE
  For each role:
  - Current skill level in relevant areas
  - Previous training and experience
  - Learning preferences and constraints
  - Time available for learning

STEP 3: IDENTIFY GAPS
  Gap = Future requirement - Current capability

  | Skill/Knowledge Area | Required Level | Current Level | Gap Size | Priority |
  |---------------------|---------------|---------------|----------|----------|
  | [Skill A]           | [1-5]         | [1-5]         | [0-4]    | [H/M/L]  |
  | [Skill B]           | [1-5]         | [1-5]         | [0-4]    | [H/M/L]  |

STEP 4: DESIGN INTERVENTIONS
  Match gap size to intervention type:
  - Gap 1: Job aids, quick reference guides
  - Gap 2: E-learning, self-paced modules
  - Gap 3: Instructor-led training, workshops
  - Gap 4: Immersive programs, coaching, extended practice
```

### Learning Path Design

| Phase | Focus | Methods | Duration | Outcome |
|-------|-------|---------|----------|---------|
| **Awareness** | Why the change, big picture | Intro video, town hall, FAQ | 1-2 hours | Understands context and rationale |
| **Conceptual** | New concepts, frameworks | E-learning, reading, webinar | 2-4 hours | Grasps new mental models |
| **Procedural** | Step-by-step how-to | Instructor-led, demos, job aids | 4-8 hours | Can follow new procedures with support |
| **Applied** | Practice in safe environment | Sandbox, simulation, role-play | 4-16 hours | Can perform independently |
| **Proficient** | Real-world mastery | On-the-job coaching, peer support | 2-8 weeks | Fluent in new way of working |

**Design principles**:
- Space learning over time (not a one-day dump)
- Provide practice opportunities before go-live
- Make job aids available at point of need
- Pair training with immediate application
- Offer multiple modalities (visual, auditory, hands-on)

## Common Change Scenarios

### Technology / System Implementation

| Phase | Key Actions | Common Pitfalls |
|-------|-------------|-----------------|
| **Planning** | Stakeholder mapping, impact assessment, pilot group selection | Treating it as an IT project instead of a people project |
| **Design** | User involvement in design, data migration planning, integration testing | Designing for the "happy path" only |
| **Training** | Role-based training, sandbox practice, super-user development | Training too early (forget by go-live) or too late (no time to practice) |
| **Go-Live** | Hyper-care support, rapid issue resolution, parallel running if needed | Big bang when phased would be safer |
| **Stabilization** | Bug fixes, additional training, process refinement | Pulling support team too early |

### Organizational Restructuring

| Phase | Key Actions | Common Pitfalls |
|-------|-------------|-----------------|
| **Design** | Future state org design, role mapping, talent assessment | Designing structure around people instead of strategy |
| **Announcement** | Clear communication of rationale, timeline, and impact | Leaks before official communication, ambiguity about roles |
| **Transition** | Role transitions, reporting changes, team formation | Neglecting the "neutral zone" emotional experience |
| **Integration** | Team building, process alignment, culture integration | Assuming the new structure will work without active integration |
| **Optimization** | Performance review, adjustments, lessons learned | Declaring victory before new structure has proven effective |

### Process Transformation

| Phase | Key Actions | Common Pitfalls |
|-------|-------------|-----------------|
| **Assessment** | Current state mapping, pain point identification, benchmarking | Redesigning without understanding current state |
| **Design** | Future state process design, automation opportunities, KPI definition | Over-engineering, designing for perfection |
| **Pilot** | Small-scale test, measure results, iterate | Skipping pilot due to time pressure |
| **Rollout** | Phased deployment, training, parallel running | Rolling out to everyone at once |
| **Optimization** | Continuous improvement, metric tracking, refinement | Treating "go-live" as "done" |

### Culture Change

| Phase | Key Actions | Common Pitfalls |
|-------|-------------|-----------------|
| **Define** | Articulate desired culture, identify behavioral indicators | Defining culture in vague terms ("be innovative") |
| **Model** | Leadership behavior change, visible actions, storytelling | Leaders espousing new values while exhibiting old behaviors |
| **Embed** | Align systems (hiring, promotion, reward, recognition) | Expecting culture to change without changing systems |
| **Reinforce** | Celebrate aligned behaviors, address misalignment | Punishing failure in a culture that claims to value experimentation |
| **Sustain** | Ongoing measurement, narrative reinforcement | Expecting culture change in months (it takes years) |

**Key insight**: Culture change is the hardest type of change. It is not a project with an end date; it is a sustained leadership commitment. Attempting culture change without aligning systems (rewards, promotions, hiring criteria) is futile.

### M&A Integration

| Phase | Key Actions | Common Pitfalls |
|-------|-------------|-----------------|
| **Day 1 Readiness** | Essential operations continuity, communication plan, immediate decisions | Information vacuum creating anxiety and talent flight |
| **First 100 Days** | Quick wins, relationship building, integration planning | Moving too fast on integration before understanding the acquired culture |
| **Functional Integration** | Systems, processes, policy harmonization | "Winner takes all" approach that devalues acquired company |
| **Cultural Integration** | Blend of best practices, shared identity formation | Imposing acquirer culture wholesale |
| **Value Realization** | Synergy capture, performance tracking, adjustment | Focusing on cost synergies at the expense of revenue/talent synergies |

## Change Management Anti-Patterns

| Anti-Pattern | Description | Why It Fails | Alternative |
|--------------|-------------|-------------|-------------|
| **"Just communicate more"** | Treating resistance as a communication problem | Awareness is only Step 1 of ADKAR; knowing why doesn't mean wanting to, knowing how, or being able to | Diagnose which ADKAR element is actually the barrier |
| **The Big Bang** | Rolling out everything to everyone at once | No opportunity to learn, iterate, or build momentum | Phased rollout with pilot groups and incremental expansion |
| **Training = Change Management** | Equating training with the entire change effort | Training addresses Knowledge and Ability but not Awareness, Desire, or Reinforcement | Comprehensive approach across all ADKAR elements |
| **Absent Sponsor** | Sponsor launches and disappears | Without sustained visible leadership, change loses credibility and urgency | Sponsor roadmap with specific commitments at each phase |
| **Change Fatigue Denial** | Launching initiative #12 this year without acknowledging overload | People disengage and protect themselves by minimally complying | Change portfolio management -- sequence, consolidate, or defer |
| **Declaring Victory Early** | Celebrating go-live as success before adoption is proven | Reversion to old ways without sustained reinforcement | Define success as sustained adoption metrics, not go-live |
| **The Passive Mandate** | Senior leader says "make it happen" without personal engagement | Middle managers interpret this as optional or negotiate scope down | Active engagement: barriers removed, resources provided, follow-up on progress |
| **Ignoring the Middle** | Focusing on executives and front-line, skipping middle managers | Middle managers are the critical translation layer; unsupported, they become the bottleneck | Equip middle managers first -- they are the change multiplier |
| **Copy-Paste Framework** | Applying the same change playbook regardless of context | Every change has unique organizational context, culture, and history | Tailor approach to change type, scale, organizational maturity, and readiness |
| **Metrics Without Action** | Tracking adoption dashboards without acting on what they show | Data without response is decorative, not functional | Define thresholds and intervention protocols for every metric |
| **One-Way Communication** | Broadcasting information without listening mechanisms | People feel unheard, resistance goes underground | Build feedback loops into every communication channel |
| **The Silver Bullet Tool** | Believing a change management platform or methodology solves everything | Tools are enablers, not solutions; people do the work | Use frameworks as thinking aids, not paint-by-numbers processes |

## Framework Selection Guide

| Situation | Recommended Framework(s) | Rationale |
|-----------|-------------------------|-----------|
| Individual adoption tracking | ADKAR | Diagnoses exactly where each person is stuck |
| Large-scale organizational initiative | Kotter's 8-Step | Provides sequenced leadership playbook |
| Quick conceptual framing | Lewin (Unfreeze-Change-Refreeze) | Simple, intuitive, communicates easily |
| Complex multi-dimensional change | McKinsey 7-S | Ensures all organizational elements are aligned |
| Emotional/identity-laden change | Bridges' Transition Model | Addresses psychological dimension others miss |
| Combination | ADKAR + Kotter | Kotter for organizational leadership; ADKAR for individual tracking |

## Quick Reference: Change Readiness Checklist

Before launching any change initiative, verify:

- [ ] Business case is clear and compelling (why now?)
- [ ] Executive sponsor is identified and actively engaged
- [ ] Stakeholder map is complete with strategies per quadrant
- [ ] Impact assessment covers process, technology, people, structure, and culture
- [ ] Communication plan addresses all audiences with appropriate framing
- [ ] Resistance sources identified with mitigation strategies
- [ ] Training plan matches competency gaps with appropriate interventions
- [ ] Champion network is recruited and equipped
- [ ] Success metrics defined with leading and lagging indicators
- [ ] Risk register includes change-specific risks with mitigations
- [ ] Change capacity assessed (how much change can the org absorb right now?)
- [ ] Feedback mechanisms are in place and monitored
