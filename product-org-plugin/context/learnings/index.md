## Learning Entry Format

```markdown
### L-[NNN]: [Learning Title]

**Source**: [Retrospective/Outcome Review/Decision Audit] for [ID]
**Date**: [When captured]
**Tags**: [category tags]
**Confidence**: High / Medium / Low

**The Learning**:
[One clear sentence describing what we learned]

**Context**:
[Brief background on what led to this learning]

**Evidence**:
[What data or outcomes support this learning]

**Application**:
[When/how to apply this learning in the future]
```

## How Learnings Flow

1. `/retrospective` generates learnings from project reviews
2. `/outcome-review` validates/invalidates assumptions, generating learnings
3. `/decision-quality-audit` identifies decision patterns, generating learnings
4. All learnings are extracted and added here via `/context-save`
5. `/relevant-learnings` searches this index for applicable wisdom
