# V2V Skill Map

Quick reference mapping all skills to V2V phases.

## Phase 1: Strategic Foundation (Input Phase)

**Purpose**: Research, analyze, and document the strategic context

| Skill | Purpose |
|-------|---------|
| `/strategic-intent` | Document strategic intent and direction |
| `/vision-statement` | Draft product vision statements |
| `/market-analysis` | Create comprehensive market analysis |
| `/competitive-landscape` | Create comprehensive competitive analysis report |
| `/competitive-analysis` | Structure a competitive analysis |
| `/market-segment` | Define a market segment |

**Exit Criteria**: Strategic intent documented, market understood, competitive position clear

---

## Phase 2: Strategic Decisions ("Commercial Filter")

**Purpose**: Make go/no-go decisions with explicit assumptions

| Skill | Purpose |
|-------|---------|
| `/business-case` | Create a comprehensive business case |
| `/business-plan` | Create a complete business plan |
| `/pricing-strategy` | Create a complete pricing strategy document |
| `/pricing-model` | Design a pricing model |
| `/positioning-statement` | Create a positioning statement |
| `/decision-record` | Create a structured decision record |
| `/strategic-bet` | Formulate a strategic bet with explicit assumptions |
| `/decision-charter` | Create a Decision Interface Charter for recurring decisions |
| `/escalation-rule` | Define escalation rules for a decision area |

**Exit Criteria**: Business case approved, pricing defined, positioning clear, assumptions explicit

---

## Phase 3: Strategic Commitments ("Point of No Return")

**Purpose**: Elaborate roadmap, establish GTM, define requirements

| Skill | Purpose |
|-------|---------|
| `/product-roadmap` | Create a complete product roadmap document |
| `/roadmap-theme` | Define a roadmap theme with initiatives |
| `/roadmap-item` | Define a specific roadmap item |
| `/gtm-strategy` | Create a comprehensive go-to-market strategy |
| `/gtm-brief` | Create a go-to-market brief |
| `/launch-plan` | Create a complete product launch plan |
| `/strategy-communication` | Create strategy communication package for stakeholders |
| `/commitment-check` | Validate commitment is properly hardened |
| `/prd` | Create a complete Product Requirements Document |
| `/prd-outline` | Create a PRD outline |
| `/feature-spec` | Create a feature specification |
| `/user-story` | Write a user story with acceptance criteria |

**Exit Criteria**: Roadmap committed, GTM established, requirements defined, commitment validated

---

## Phase 4: Coordinated Execution ("The Doing" Phase)

**Purpose**: Execute across all functions in coordination

| Skill | Purpose |
|-------|---------|
| `/launch-readiness` | Launch readiness decision checklist |
| `/stakeholder-brief` | Create a stakeholder communication brief |
| `/campaign-brief` | Create marketing campaign brief |
| `/sales-enablement` | Create sales enablement package |

**Exit Criteria**: Launch executed, all functions coordinated, stakeholders informed

---

## Phase 5: Business & Customer Outcomes (Output Phase)

**Purpose**: Measure and realize customer value

| Skill | Purpose |
|-------|---------|
| `/onboarding-playbook` | Create customer onboarding playbook |
| `/value-realization-report` | Create value realization report |
| `/customer-health-scorecard` | Create customer health scorecard |

**Exit Criteria**: Value delivered, outcomes measured, customer health tracked

---

## Phase 6: Learning & Adaptation Loop (System's "Brain")

**Purpose**: Learn from outcomes and feed back into strategy

| Skill | Purpose |
|-------|---------|
| `/outcome-review` | Structure an outcome review for learning |
| `/retrospective` | Conduct structured retrospective |
| `/decision-quality-audit` | Audit recent decisions for quality |
| `/context-save` | Save decision, bet, or learning to the context registry |
| `/relevant-learnings` | Find past learnings applicable to current work |
| `/feedback-capture` | Capture and analyze product feedback |

**Exit Criteria**: Learnings captured, assumptions validated/invalidated, context updated

---

## Cross-Phase Skills

These skills operate across multiple phases:

| Skill | Purpose | Phases |
|-------|---------|--------|
| `/context-recall` | Query past decisions, bets, and learnings | All |
| `/feedback-recall` | Query past feedback by topic or theme | All |
| `/portfolio-status` | View current state of all active strategic bets | All |
| `/portfolio-tradeoff` | Structure a portfolio-level tradeoff decision | 2, 3 |
| `/handoff` | Capture context for agent-to-agent delegation | All |
| `/setup` | Initialize the Product Org plugin | All |
| `/present` | Convert a document to HTML presentation | All |
| `/qbr-deck` | Create Quarterly Business Review presentation | All |
| `/maturity-check` | Assess organizational maturity | All |
| `/pm-level-check` | Assess PM competency level | All |

## Principle Validator Skills

These skills enforce the 8 Operating Principles:

| Skill | Principle | Purpose |
|-------|-----------|---------|
| `/ownership-map` | #1 End-to-End Ownership | Map accountability across V2V phases |
| `/customer-value-trace` | #3 Customer Obsession | Validate work traces to customer value |
| `/collaboration-check` | #6 Collaborative Excellence | Validate RACI and stakeholder consultation |
| `/scale-check` | #8 Scalable Systems | Assess scalability at 2x, 10x, 100x |
| `/phase-check` | V2V Flow | Assess which V2V phase an initiative is in |

---

## Usage

### Check Phase Readiness
```
/phase-check [initiative name]
```

### Validate Before Commitment
```
/commitment-check [initiative name]
```

### Find Skills for Current Phase
1. Identify your current V2V phase
2. Reference the skill table for that phase
3. Use appropriate skills based on deliverable needed
