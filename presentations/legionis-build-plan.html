ARCHITECTURE

Gateway Architecture (Per-Team Deliverables)

Each team gateway requires two files working together. This is the critical architectural pattern that makes @finance, @legal, etc. work as invocable commands.

1. GATEWAY.md

Extension Teams/{team}-team/GATEWAY.md

Purpose: Defines routing logic, agent roster, hierarchy, cross-team delegation patterns

~190 lines per gateway

  • Team personality and principles reference
  • Adaptive Response System
  • Domain ownership map (keyword to agent)
  • Agent spawning protocol
  • Cross-team delegation patterns
  • Knowledge packs and integrations
Pattern: Follow design-team/GATEWAY.md structure

2. Gateway Skill File

Product Org OS/product-org-plugin/skills/{team}/SKILL.md

Purpose: Makes the gateway invocable via @{team} or /{team} through the Skill tool

~200-250 lines per skill file

  • Frontmatter (name, model, user-invocable: true)
  • Reference to team's GATEWAY.md
  • Spawning instructions (Task tool)
  • Integration with identity registry
Pattern: Follow skills/product/SKILL.md structure
